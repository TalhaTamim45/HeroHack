# Security and Ethics Policy

CyberQuest Ops is designed for legal cybersecurity education.

## Core rule

Think like an attacker. Act like a defender. Work only with permission.

## Allowed learning content

The platform may teach:

- Security mindset
- Scope and permission
- Web security concepts
- Network concepts
- Log analysis
- Defensive hardening
- Secure coding
- WordPress security
- Cloud security concepts
- Report writing
- Incident response

## Not allowed in public gameplay

The platform should not provide features that let users:

- Scan real public targets
- Attack third-party systems
- Steal credentials
- Deploy malware
- Phish real users
- Bypass real accounts
- Exfiltrate real data
- Abuse public infrastructure

## Safe lab rules

All practical missions should use:

- Fictional domains
- Toy apps
- Training logs
- Simulated networks
- Isolated containers
- Clear scope
- Resettable environments

## Mission design checklist

Every mission should answer:

1. What is the authorized target?
2. What is out of scope?
3. What evidence should the player collect?
4. What defensive lesson is included?
5. What fix should the player recommend?
6. What report should the player write?

## Tone

The game should feel fun and exciting, but not criminal.

Good tone:

- Investigator
- Defender
- Ethical operator
- Security engineer
- Consultant

Bad tone:

- Real-world criminal hacking
- Revenge hacking
- Targeting random people
- Credential theft outside a lab
